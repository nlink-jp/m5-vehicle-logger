## GPSモジュール: M5 GPS Module v2.1

### チップ

- **AT6668**（旧 v2.0 の AT6558 とは異なる）
- マルチコンステレーション: GPS, BDS (BD2/BD3), GLONASS, Galileo, QZSS
- デフォルト出力: **NMEA 0183**
- デフォルトボーレート: **115200**（9600 ではない）

> **重要:** 「M5 GPS Module」に関するネット上の多くの情報は AT6558 / 9600 baud を前提としている。v2.1 の AT6668 は 115200。9600 で接続するとバイナリプロトコルのような文字化けデータになる。

### ディップスイッチ

モジュール上にディップスイッチがあり、TX/RX ピンマッピングを選択可能。M5Stack Basic v2.7 のスタックコネクタ接続では以下に設定：

| スイッチ | 設定 |
|---------|------|
| TXD    | G17  |
| RXD    | G16  |
| PPS    | G36  |

### ESP32 シリアル設定

ESP32 の `Serial2.begin()` パラメータ：

```cpp
// RX pin = G16（GPSのTXDから受信）
// TX pin = G17（GPSのRXDへ送信）
Serial2.begin(115200, SERIAL_8N1, 16, 17);
```

> **注意:** M5Unified の `M5.begin()` が内部的に Serial2 を初期化する場合がある。GPS プロバイダは正しいピンとボーレートで再初期化する前に `Serial2.end()` を呼ぶ必要がある。

### アンテナ

- **外部 SMA アンテナ必須。** モジュールに内蔵アンテナはない。
- アンテナ未接続では衛星信号を一切受信できない。
- **空が見える場所**にアンテナを設置する。壁の裏や金属屋根の下は不可。窓越しでも受信可能だが、可視衛星数が制限される。

### コールドスタート時間

| 起動タイプ | 条件 | 初回fix時間 |
|-----------|------|-----------|
| コールドスタート | 初回電源投入、アルマナックなし | 5〜15分 |
| ウォームスタート | アルマナックあり、エフェメリス期限切れ | 30〜60秒 |
| ホットスタート | 直近のfix、短時間の電源断 | 1〜5秒 |

- コールドスタート中もNMEAセンテンスは出力されるが、位置フィールドは空。
- **可視衛星数**（GSVセンテンス）はfix前に確認可能。画面には `no fix N sat` と表示される。
- 測位（fix）には **4衛星以上**が必要。

### NMEAセンテンス

AT6668 がデフォルトで出力するセンテンス：

| センテンス | 内容 |
|-----------|------|
| `$GNGGA` | 位置、高度、衛星数、fix品質 |
| `$GNGLL` | 位置、時刻 |
| `$GNGSA` | DOP値、使用中衛星 |
| `$GPGSV` / `$BDGSV` / `$GLGSV` | 可視衛星情報（コンステレーション別） |
| `$GNRMC` | 位置、速度、方位、日付 |
| `$GNVTG` | 対地コース・速度 |

> 注: AT6668 は大半のセンテンスに `GN` プレフィックス（マルチコンステレーション）を使用し、GSV にはコンステレーション別プレフィックス（`GP`, `BD`, `GL`）を使用する。

### トラブルシューティング

| 症状 | 原因の可能性 | 対処 |
|------|------------|------|
| データが全く来ない（0バイト） | ボーレートかピンが間違い | 115200 を確認、ディップスイッチ確認 |
| バイナリ/文字化けデータ | ボーレート不一致 | 115200 に変更 |
| NMEA受信するが `no fix searching...` | アンテナ未接続または遮蔽 | アンテナ接続、屋外へ |
| `no fix N sat` だがfixしない | 衛星不足（4未満） | 待つ、アンテナ配置改善 |
| `M5.begin()` 前だけデータ受信 | Serial2 競合 | 再初期化前に `Serial2.end()` |

## M5Stack Basic v2.7

### v2.6 との主な違い

- スタックコネクタの UART ピンが旧ドキュメントと異なる場合がある。実機で要確認。
- **PSRAM 非搭載** — `M5Canvas::createSprite(320, 240)`（150KB）は確保に失敗する。スプライトではなく、背景色付きテキスト上書きによる直接LCD描画を使う。

### M5Unified ライブラリ

- レガシー M5Stack ライブラリではなく **M5Unified** を使用する。
- レガシー M5Stack ライブラリ（v0.4.6）は ESP32 ボードパッケージ v3.x と非互換（`rom/miniz.h` が存在しない）。
- M5Unified API の違い：
  - `M5.Lcd` → `M5.Display`
  - `M5.begin(lcd, sd, serial, i2c)` → `auto cfg = M5.config(); M5.begin(cfg);`
  - `TFT_eSprite` → `M5Canvas`（ただし PSRAM なしでは大きなスプライト不可）
  - SD は `M5.begin()` で自動初期化

### ESP32 ボードパッケージ

- レガシー M5Stack ライブラリ使用時: **v2.1.2**
- M5Unified 使用時: **v3.3.7 以降**

### クラス名の競合

ESP32 Arduino Core v3.x に `NetworkManager` クラスが定義されている。コード内でこの名前を使うと再定義エラーになる。本プロジェクトでは `WifiManager` に改名して回避。
